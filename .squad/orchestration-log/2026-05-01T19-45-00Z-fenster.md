### Production Bug Fix (Stop-the-Bleed)
**Endpoint:** `POST /api/finances` (currently broken on Vercel)  
**Migration:** Server Action (`app/current-finances/actions.ts`) → Supabase direct  
**Shape:** 
- Server Action fetches household_id from `user_profile.default_household_id`
- Upserts `finance_snapshots` (PK: household_id, date)
- RLS enforces via `is_household_writer(household_id)`
- Returns `{ success: boolean, error?: string }`
- Client shows inline error banner (no more `alert()`)

**Effort:** M-size (2-4 hours) — Server Action + error UX + unit/E2E tests

### Decision Criteria
- **Use Server Action:** Mutation with business logic, data write, avoid exposing Supabase client, need server context
- **Use Direct Supabase:** Read-only, real-time subscriptions, optimistic UI, user-driven params
