# Round 8 — Currency Display Final Fix (2026-05-12)

## Trigger
User pasted 20-position Leumi IRA comparison table; demanded definitive cross-currency fix. Pre-Round-8 ground truth: BARC, LUMI, QQQI, CLIS positions showed systemic 100× errors or currency-rate fallback bugs across GBP/ILS/USD.

## Phase 1: Audit fan-out (parallel)

**Keaton-4 (Lead)** → Authored 3-currency architectural contract
- Definitive spec: eliminate sub-unit storage, use major canonical currency (ILS/GBP/USD) everywhere, `currency` = ISO codes only
- Root cause analysis: stale Docker container pre-PR-#420, repeated 8 rounds of reactive patches
- **Outcome:** Deferred full migration to Issue #423 in favour of pragmatic Phase 2 fixes

**Hockney-14 (Backend)** → Deep DB audit: 30 Leumi positions
- Three-way comparison: spreadsheet ground truth ↔ DB state ↔ UI display
- **Root cause identified:** Stale Docker container `33fd12cab77e` (pre-PR-#420) runs old worker code that:
  - Writes `market_value = qty × mark_price` without `÷100` for LSE positions → GBP 100× inflated
  - Computes `dividend_yield` via 100× wrong formula instead of deterministic rate-based
  - Overwrites post-migration corrections on next daily run (06:59 UTC)
- **Verification protocol:** Compare container Created timestamp vs. latest worker commit; rebuild if stale

**Fenster-11 (Frontend)** → Display surface inventory audit
- 9 surfaces enumerated across `/trading/accounts`, `/dividends`, `/summary`, `/bonds`, `/ladder`
- **Key finding:** 0 out of 9 correctly handle all 3 currencies
  - GBP missing from CURRENCY_RATES → ~3.6× undervalued in USD aggregates
  - GBP mark_price ÷100 guard absent on most surfaces
  - No currency-aware FX conversion on aggregate totals

## Phase 2: Pragmatic fixes (parallel)

**Hockney-15 (Backend)** → PR #425 (container rebuild + Yahoo refresh + migration)
- Rebuilt Docker container: `33fd12cab77e` → `f524b85d7383`
- Ran Yahoo refresh (297 positions refreshed, 17 unchanged, 7 errors)
- Applied migration `20260512170000_sync_market_value_from_local.sql` to correct DB values post-refresh
- **Decision recorded:** Container MUST be rebuilt after every worker-code PR, before next scheduled run
- **Outcome:** PR #425 OPEN (awaiting user merge)

**Fenster-12 (Frontend)** → PR #424 (GBP display + QQQI trustworthiness gate)
- Extended ÷100 guard to GBP: `isSubUnit(c) = c === 'ILA' || c === 'GBP' || c === 'GBp'`
- Added `GBP: 4.6` to CURRENCY_RATES (closes 3.6× undervaluation bug)
- Defensive normalisation in formatCurrency: ILA→ILS, GBp→GBP
- **QQQI TTM yield trustworthiness gate:** MIN_TTM_PAYMENTS_FOR_TRUST=3, MAX_DAYS_SINCE_LAST_PAYMENT=60
  - Only use DB `dividend_yield` if ≥3 payments AND ≤60 days since last
- Test results: 646 regression tests pass ✅
- **Outcome:** PR #424 OPEN (awaiting user merge)
