## Design Principles

1. **Decimal precision:** All monetary calculations use `decimal.Decimal` (converted to float at serialization boundary only)
2. **Pure functions:** No DB access, no network I/O, no side effects — framework-agnostic
3. **Composable:** Pydantic I/O models for composite functions; standalone functions for individual metrics
4. **Technical indicators:** List[float] interface (not pandas) for framework independence
5. **Support/Resistance:** Pivot-point clustering with configurable tolerance
