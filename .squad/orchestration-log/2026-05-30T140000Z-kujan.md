## Work Summary

**Title:** Dynamic Migration Discovery for CI Fallback Path
**Issue:** P1 bug — hardcoded migration allowlist in Supabase CI workflow silently skipped new migrations

### Problem Discovered

The `.github/workflows/supabase-migrations.yml` db-url fallback path used `apply_expense_pipeline_directly()` with a hardcoded array of 2 migration filenames. New migrations were silently skipped.

**Impact:** PR #489 added Transportation migration (`20260530055800_add_transportation_category.sql`), but workflow never applied it. Prod remained at 35 categories instead of expected 39.

### Solution Implemented

Replaced hardcoded allowlist with **dynamic discovery** via new `apply_pending_migrations_directly()` function:
1. List all `*.sql` files in `supabase/migrations/` (sorted)
2. Query prod `supabase_migrations.schema_migrations` for applied versions
3. Apply only unapplied migrations via psql (idempotent)
4. Verify via `verify_expense_tables` schema check
