# E2E Test Suite — Trading Journal

## Architecture

```
e2e/
  smoke/          P0 smoke tier — fast, no auth, catch catastrophic regressions
    home.spec.ts          / → /summary redirect
    settings.spec.ts      /settings auth-guard (EXPECTED FAIL pre-Fenster PR)
    holdings.spec.ts      /holdings auth-guard (EXPECTED FAIL pre-Fenster PR)
    healthcheck.spec.ts   app boots, no JS errors, assets load
  flows/          P0/P1 flow tier — authenticated happy paths
    root.spec.ts          authenticated / → /summary
    current-finances.spec.ts  /current-finances P0 flow
    plan.spec.ts          /plan P0 flow
    summary.spec.ts       /summary P0 flow
  auth/           P1 auth tier (future: login/logout/session flows)
  rls/            P2 RLS tier (future: household isolation via E2E)
  fixtures/
    admin.ts      Service-role Supabase client + user lifecycle helpers
    auth.ts       Playwright fixtures: authenticatedUser, householdOwner
  scripts/
    cleanup-stale-users.ts  Delete e2e_* users older than 1h
  README.md       (this file)
  BASELINE.md     Smoke baseline snapshot (generated by CI run)
```

## Quick-start

```bash
# 1. Copy env file (dev Supabase project)
cp /path/to/coord/apps/frontend/.env.development.local apps/frontend/.env.local

# 2. Install dependencies
cd apps/frontend && npm install

# 3. Start dev server (background)
npm run dev &

# 4. Run smoke suite
npm run test:e2e -- e2e/smoke/ --reporter=list

# 5. Run flow suite (requires Supabase creds in .env.local)
npm run test:e2e -- e2e/flows/ --reporter=list

# 6. Run everything
npm run test:e2e
```

## Scripts

| Script | What it does |
|--------|-------------|
| `npm run test:e2e` | Run all Playwright specs (tests/ + e2e/) |
| `npm run test:e2e:headed` | Run with browser visible |
| `npm run test:e2e:ui` | Interactive Playwright UI |
| `npm run test:e2e:cleanup` | Delete stale e2e_* test users |

## Environment variables

| Variable | Required | Description |
|----------|----------|-------------|
| `NEXT_PUBLIC_SUPABASE_URL` | Yes (flows only) | Dev Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Yes (flows only) | Anon key |
| `SUPABASE_SERVICE_ROLE_KEY` | Yes (flows only) | Service-role key for admin.ts |
| `BASE_URL` | Optional | Override default `http://localhost:3000` |
| `SUPABASE_E2E_ALLOW_PROD` | No | Set `true` to bypass prod-guard (⚠️ dangerous) |

## Throwaway user pattern

Each `authenticatedUser` / `householdOwner` fixture creates a fresh Supabase user per test:

- Email format: `e2e_<unix-ms>_<4char-rand>@example.com`
- Password: `E2eTestPass!1`
- Deleted in `afterEach` automatically
- Stale users (>1h) can be purged with `npm run test:e2e:cleanup`

## Prod-guard

`admin.ts` checks the Supabase URL ref slug for `dev/stag/test/local/preview/sandbox` hints
before allowing test-user creation. If the URL looks like production, the test fails with a
clear error. Override with `SUPABASE_E2E_ALLOW_PROD=true` only if you are certain.
